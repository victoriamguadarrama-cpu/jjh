# Explanation.md

## Requirement 1: React/Vite
Built with Vite using the React template. See "frontend/vite.config.js" and "frontend/package.json"....all future references are in the frontend file unless otherwise indicated.

## Requirement 2: Professional Styling
Every component has its own CSS file with consistent dark color palette, Cormorant Garamond typography, hover transitions, and responsive layout. Nothing is unstyled. (Most styling code was generated by Claude)

## Requirement 3: Components in src/components
All components live under src/components, they are organized into subfolders: About, Cart, Home, Layout, Menu and Shop.

## Requirement 4: Standard Layout
Layout.jsx wraps every single page through App.jsx. TopBar, NavBar, page content, and Footer render on every route.

## Requirement 5: Layout Components in src/components/Layout
TopBar.jsx, NavBar.jsx, Footer.jsx and Layout.jsx make up the Layout components and are stored in src/components/Layout.

## Requirement 6: JSX Compliant
I use className throughout, self-closing tags when needed, and there are no stray HTML attributes. Standard JSX compliance across all files.

## Requirement 7: 3 Layout Components
TopBar.jsx for the logo and contact bar at the top, NavBar.jsx for the navigation links and live cart count, and Footer.jsx with all footer information. 

## Requirement 8: Layout Components Renders the 3
Layout.jsx renders all three components and uses children (the prop, not humans) to slot in whatever page is active.

## Requirement 9: External Backend
JSON server backend in the backend folder. To start, open the backend in vs code, npm install, npm start. It will run on port 3001. Full detail in backend/README.md.

## Requirement 10: Dynamic Content from API
Shop.jsx and ProductDetail.jsx both fetch live product data from the backend-generated endpoint localhost:3001/products. Checkout.jsx POSTs completed orders in the backend db.json file.

## Requirement 11: Dynamic List with Reusable Components
Shop.jsx maps over the products array and renders a ProductCard for each one. MenuSection.jsx maps over section.items to render each menu item button.

## Requirement 12: Props
ProductCard receives the full product object as a prop. MenuSection recieves sectionKey and section as props from Menu.jsx. 

## Requirement 13: Multiple Pages
Seven pages total, routed via window.location.pathname in App.jsx. Those pages are Home, Shop, ProductDetail, Menu, About, Cart, and Checkout.

## Requirement 14: URL Parameter-Driven Page
App.jsx uses a regex (generated by Claude: path.match(/^\/shop\/(\d+)$/)) to pull the product ID out of the URL and passes it as productId to ProductDetail.jsx. This then finds the right product, either from sessionStorage cache or a fresh API fetch.

## Requirement 15: children Prop
Layout.jsx accepts and renders the children prop, which is how every page component gets displayed inside the shared layout.

## Requirement 16: Context
CartContext.jsx in src/store manages shared state for both the t-shirt cart and the catering menu selections. It exposes derived data functions including getCartTotal, getCartCount, getMenuTotal, getMenuSelectionsByCategory (this filters selections by category prefix). The context is consumed in MenuSection, OrderSummary, ProductDetail, Cart, Checkout, and NavBar.

## Requirement 17: sessionStorage Caching
Before fetching from the API, Shop.jsx checks sessionStorage.getItem("opc-products"). If it's there, it uses it. ProductDetail.jsx does the same to ensure there are no duplicate calls.

## Requirement 18: Frontend Data Processing
Shop.jsx computes average price and price range from the raw product array before rendering. OrderSummary.jsx computes a running per-person cost estimate from selected menu items. Cart.jsx and Checkout.jsx both calculate the cart total from item prices and quantities on the frontend.

## Requirement 19: Additional Meaningful Component
OrderSummary.jsx is my additional component. It pulls menuSelections, getMenuTotal, and clearMenuSelections from context, computes a formatted per-person cost estimate, and builds a pre-filled "mailto:" link with the full list of selected items, and pricing, so the user can send their catering inquiry directly to the coordinator with one click!!!

## Requirement 20: This File
Congrats on making it all the way through!

## Architectural Decision
I put CartContext in src/store instead of inside src/components because it isnt' a UI component, its a shared application state. Keeping it separate made the distinction more explicit. Components handle rendering, and the store ahndles data. It also made it easier to find and reason about as the app grew, AND (according to Claude) this mirrors how state management is typically structured in larger React projects.
